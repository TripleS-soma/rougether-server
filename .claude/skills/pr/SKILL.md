# PR

브랜치·커밋·PR·리뷰 규칙의 정본은 저장소 루트의 **CONTRIBUTING.md**다. 이 스킬은 규칙을 재서술하지 않는다 — **먼저 CONTRIBUTING.md를 읽고** 그대로 따른다. 대상 작업은 전달받은 인자(PR 제목 또는 작업 설명) 또는 현재 변경 내용.

## 절차

1. CONTRIBUTING.md를 읽는다.
2. **브랜치** (§1): 현재 브랜치를 확인하고, `main`이면 규칙에 맞는 새 브랜치를 만든다.
3. **커밋** (§2, §6): Conventional Commits로 커밋한다. 실수 커밋·의미 없는 중간 커밋이 너무 많으면 정리한다.
4. **검증** (§7): `./gradlew test` 통과를 확인한 뒤 push한다.
5. **리뷰 필수 여부 판단** (§5): 리뷰 필수에 해당하면 PR 본문에 명시하고 셀프 머지하지 않는다.
6. **PR 생성** (§3, §4): 제목은 커밋과 같은 형식, 본문은 §4의 템플릿·체크리스트를 따른다. API 계약이 바뀌었으면 §4의 **API 변경 섹션**(🔔)을 본문 맨 아래에 추가한다 — 섹션에 멘션은 넣지 않는다(머지 시 Action이 댓글로 알림).

## Claude 전용 규칙

- `/plan`·`/run-plan`으로 진행한 작업이면 본문에 `closes #<구현 이슈번호>`를 넣는다.
